# CoinCast 3D 场景贴图（工程约定）

以下文件名与尺寸为 **工程约定**；生成后放入 `textures/` 并覆盖同名文件即可（`PhysicsCoinsScene` 已通过 `expo-three` / `loadAsync` 加载）。

---

## 0. 全局硬性要求（违反则验收不通过）

- **禁止水印与装饰**：不得出现 AI 工具角标、四角星、签名、URL、二维码等非卦象内容。
- **正方形与 POT**：所有贴图 **宽高相等**，边长为 **1024** 或 **2048**（2 的幂）。**禁止** 1408×768 等非 POT、非正方形尺寸（移动端 GPU 缩放与圆柱 UV 会出问题）。
- **文件大小**：单张 PNG 建议 **≤ 500KB**（可用压缩或略降细节）；当前工程内若单张 > 1.5MB 应重新导出。
- **色彩空间**：反照率贴图为 **sRGB**；法线为 **OpenGL 法线**（紫粉色调），法线贴图 `colorSpace = NoColorSpace`（由加载代码处理，导出时用标准法线即可）。

## 1. 木案桌面（必做）

| 文件名   | `altar-wood-albedo.png` |
| -------- | ----------------------- |
| 尺寸     | **1024×1024** 或 **2048×2048** |
| 格式     | PNG，8-bit，sRGB        |
| 内容     | **四方连续无缝**（tileable），俯视旧木案木纹；**不要**在四条边放明显木结或高光条 |
| 色相     | 暖褐、偏 `#2a2218`～`#3d3428`，与铜钱暖金同系，避免发灰发青 |
| 对比     | **整体偏低对比**，细节靠细微明暗，避免强条纹（repeat 会露接缝） |

**验收**：在 PS / Krita 中对 albedo 做 **Offset 50%**，接缝处不可见跳变或明显明暗圈。

### 可选：`altar-wood-normal.png`

| 文件名 | `altar-wood-normal.png` |
| ------ | ----------------------- |
| 尺寸   | 与 albedo **完全相同** |
| 格式   | PNG，OpenGL 法线       |
| 内容   | 与 albedo 木纹走向一致，强度中等（不要夸张鼓包） |

## 2. 铜钱两面（必做，阴阳可读）

推荐 **方案 A**（与当前工程一致：圆柱 **+Y 盖** = 阳，`−Y` 盖 = 阴）。

### 阳面（花面）— `coin-yang-cap.png`

| 项目 | 要求 |
| ---- | ---- |
| 尺寸 | **1024×1024**（或 2048²） |
| 构图 | 古钱：**外圆 + 中心方孔**，纹样在环内；圆占画面约 **78%～85%** 直径，四周留均匀边距 |
| 气质 | 偏「花面」：**略亮、略疏、略开放**（可延续星芒/放射意象） |
| 四角 | **不透明**，底色接近木案中灰 `#3a3228`～`#4a4036`，避免白底在 3D 里切出硬圈 |
| 线稿 | 略粗于纯印刷稿，**适配手机小屏** |
| 明度 | 主区域建议 **≥ `#c8b090`**（与阴面拉开档） |

### 阴面（字面）— `coin-yin-cap.png`

| 项目 | 要求 |
| ---- | ---- |
| 尺寸 | 与阳面相同 |
| 构图 | 同结构：外圆 + 方孔 |
| 气质 | 偏「字面」：**略暗、略密、略收**（云纹/曲笔等抽象纹样）；**整体比阳面暗 1～2 档**，仍保持暖铜色底，不要冷灰 |
| 文字 | **不要**可辨认的杜撰年号或商用字体汉字（版权与 README 历史约定一致）；用抽象云纹、回纹、符号化笔画即可 |
| 明度 | 主区域建议 **≤ `#7a6548`**，与阳面形成清晰明度差 |

**小屏验收**：缩略到 **200×200 px** 仍能一眼分辨阴阳（靠明度 + 纹样疏密，不靠细线）。
