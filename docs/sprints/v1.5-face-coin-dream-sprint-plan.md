# Part 1 · Coincast (W5)

## Scope (per ADR-0017 §"Coincast")

I-Ching study tool + Hexagram Journal. Free unlimited divinations + Pro for journal/study mode.

**Anti-spam framing**: NOT "fortune-telling app"; positioning as **易经 study companion** with classical 王弼 / 朱熹 commentary depth. Real coin physics OR 40-stalk method (both options for accuracy purists).

## Sprint C.1 · Coin physics + 40-stalk + 64-hexagram core

**Goal**: deliver both classical divination methods with deterministic algorithm.

### Deliverables

1. `packages/coincast-engine/` — new workspace package
   - `coinThrow(): HexagramLine[]` — physics-simulated 3-coin toss × 6 (with classical 老阴/老阳/少阴/少阳 mapping)
   - `yarrowStalk(): HexagramLine[]` — 40-stalk classical 周易筮法 simulation (with the 18-pass procedure)
   - `interpretHexagram(hex: number, changingLines: number[]): { original: Hexagram, transformed: Hexagram | null }`
2. 64 hexagram data:
   - Name + 卦象 visual + 卦辞 (Wang Bi commentary) + 爻辞 × 6 (Zhu Xi commentary)
   - Multi-locale (4 languages)
   - Reference data in `apps/coin-cast-app/data/hexagrams/`
3. `apps/coin-cast-app/app/(tabs)/divination.tsx`:
   - Method selector: Coin (fast) / Yarrow (classical, animated 18-pass)
   - Question input (optional, for journal context)
   - Output: hexagram + changing lines + transformed hexagram (if changes)
   - Tap each line → classical commentary

### Effort: ~5-6 person-days. Algorithm + data.

### Risk: Yarrow stalk animation may feel slow on first load. Mitigate with skip option.

## Sprint C.2 · Hexagram Journal + study mode (Pro)

**Goal**: track consultations over time + structured 易经 learning path.

### Deliverables

1. `apps/coin-cast-app/app/(tabs)/journal.tsx`:
   - List of all consultations (Pro)
   - Each entry: date, question, hexagram, transformed, user notes
   - Search + filter by hexagram, by changing-line pattern
   - Compare entries: "show all my 乾 entries this year"
2. Study mode:
   - 64-hexagram learning path (8 weeks structured curriculum)
   - Each hexagram: structural analysis + classical commentary + worked examples
   - Progress tracking per user
3. `apps/hexastral-api/src/routes/coincast-journal.ts` — backend persistence
4. Family / sharing: optional sharing of selected consultations to a study group

### Effort: ~5-6 person-days

### Risk: Journal feels under-used without questions. Mitigate with onboarding "ask your first question" guided flow.

# Part 2 · FaceRead (W6)

## Scope (per ADR-0017 §"FaceRead")

Classical Physiognomy Reference + Family Face Lineage. Track features over time + family resemblance + historical figure comparison.

**Anti-spam framing**: NOT "palm reading," NOT predictive. Position as **facial analysis utility** with classical 神相 reference layer.

## Sprint F'.1 · Photo capture + 12-宫 feature analysis core

**Goal**: photo input → structured facial 12-宫 analysis.

### Deliverables

1. `apps/face-oracle-app/components/PhotoCapture.tsx`:
   - Front-facing camera with face-detection guide overlay
   - Photo retained on-device by default; only feature features sent to backend
2. `apps/face-oracle-app/components/TwelvePalacesView.tsx`:
   - 12-宫 sector overlay on captured photo
   - Tap each palace → classical 神相 reference
   - Multi-locale labels
3. Backend: existing `apps/hexastral-api/src/routes/physiognomy/face-features.ts` route extends
   - Gemini Vision returns structured features (no narrative)
   - 12-宫 mapping deterministic from feature positions
4. Privacy framing: explicit "photo stays on device unless you opt in to sync"

### Effort: ~5-6 person-days

### Risk: Gemini Vision feature reliability varies by photo quality. Mitigate with guidance overlay + retake prompt.

---

## Sprint F'.2 · Family Face Lineage + time-series tracking (Pro)

**Goal**: family resemblance tree + face change tracking.

### Deliverables

1. Family face lineage:
   - Multi-person face capture (parents / siblings / children, with consent)
   - Cross-person feature comparison (same chin, same nose, etc.)
   - Tree visualization (similar to MingPan family lineage)
2. Time-series tracking:
   - Re-capture every 3-6 months
   - Track changes in features over time
   - Insights: "Your forehead has shown XYZ change since last year"
   - Privacy: time-series photos stay on-device (or iCloud encrypted)
3. Anti-spam: NO age/health predictions — just structural feature change tracking

### Effort: ~5-6 person-days

### Risk: face change detection might raise medical/health concern flag (1.4.1). Mitigate with strong disclaimer: "NOT medical assessment, classical facial study only."

---

## Sprint F'.3 · Historical figure comparison + widget + watch

**Goal**: compare your face to 20 historical figures + widget + watch.

### Deliverables

1. Historical figures database:
   - 20 figures across 朝代 (汉 / 唐 / 宋 / 明 / 清)
   - Each: documented facial features (from portraits or classical descriptions)
   - Educational context: how their facial features manifested in their life
2. Comparison view:
   - "You share 厚耳 with 苏轼" / "You share 长眉 with 孔子" (per classical analyses)
   - Educational note: no predictive claims
3. Widget × 3:
   - Small: today's recorded face stats (changes from last capture)
   - Medium: next family member to add + most-similar historical figure
   - Lock-Screen: time-series snapshot countdown ("next capture in N days")
4. Watch complication: photo capture reminder

### Effort: ~5-6 person-days

### Risk: historical figure facial features are subjective. Mitigate with academic citations for each interpretation.

# Part 3 · DreamRead (W7)

## Scope (per ADR-0017 §"DreamRead")

Dream Journal + Classical 解梦 Reference + Pattern Recognition. Free journal + Pro for pattern dashboard + Pro for psychological framework.

**Anti-spam framing**: NOT "fortune-telling dream interpretation." Position as **dream journal** with classical/psychological reference depth.

## Sprint D.1 · Dream journal core + voice + symbol tagging

**Goal**: dream journal with voice + tags.

### Deliverables

1. `apps/dream-oracle-app/app/(tabs)/journal.tsx`:
   - Quick entry: voice recording + text + symbol tags
   - Common symbols (water / mountain / flying / falling / etc.) with classical mapping
   - Date + sleep cycle / time correlation
2. `apps/dream-oracle-app/components/DreamTagPicker.tsx`:
   - 周公解梦 24-category tag system
   - Multi-select per dream
   - Multi-locale labels
3. Backend: extends existing `dream-oracle-app` routes for persistence

### Effort: ~5-6 person-days

### Risk: voice recording UX must be polished — half-asleep users won't read instructions.

## Sprint D.2 · Pattern dashboard (Pro)

**Goal**: dream pattern recognition over time.

### Deliverables

1. Pattern dashboard:
   - Recurring symbols
   - Lucid dream frequency
   - Sleep quality correlation (if HealthKit access granted)
2. 周公解梦 24-category mapping:
   - Each category → classical 周公 commentary

3. Modern psychological framework:

### Effort: ~5-6 person-days

### Risk: pattern detection needs ≥ 30 dreams for meaningful recurrence. Mitigate with onboarding setting expectations + Pro paywall after 7 dreams (not immediately).
