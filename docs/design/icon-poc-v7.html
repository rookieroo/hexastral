Astral Minimal · 设计语言探索

天体极简

由 fate-app 那枚渐变黑球反推的一套语言：深黑画布 · 渐变天体为母题 · 单一光向 · 极简留白 · 近灰度 + 一缕暖金 · 轻量字。 抛开 hexastral-token 与金石/古字调性——这里靠光与体积而非笔画与蚀刻。每个 App 不是"另画一个图"，而是同一颗球的不同天象（相位 / 蚀 / 环 / 双星 / 弦月）。

01

APP 图标 · 天体家族

同一颗球，八种天象。差异来自光与相位，不靠颜色喧哗（保留参考图的灰度克制）；命/相/卜的归类靠排布与文案，不入图标。

02

APP 内图标 · 细线

球是品牌/英雄元素；功能图标走1.5px 细线、圆角、单色，安静不抢戏。

03

分享卡 · Reading Report · 轮播

同一语言铺到三种界面：分享卡(社媒) · Reading Report(报告) · 轮播 card list(首屏今日)。深黑 + 一颗球 + 轻字 + 发丝分隔线。

这套相对 v5/v6 的取舍：共性——深黑、克制、神秘、"Astral"贴名号；强项——更现代/国际、可缩放(球在任何尺寸都成立)、铺界面成本低、与 Co-Star/Oura/Calm 一档的高级感； 代价——丢了汉字的文化纵深与"金石"手感，八图差异偏弱(靠相位区分，需谨慎设计或加极淡色相)。
建议：可把天体极简定为产品级 UI 语言(token / 内图标 / 卡片 / 报告)，把古字(v5)留作品牌印记(启动页、关于页、实体周边)——两者并不冲突：一个是"皮肤"，一个是"图腾"。
