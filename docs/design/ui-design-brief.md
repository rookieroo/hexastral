## 3. 设计 Tokens

**色**
```
--bg        #0A0B10   画布（深夜）
--surface   #131722   卡片
--surface-2 #1A2030   浮起卡 / 输入
--line      rgba(236,231,218,.09)   发丝分隔
--line-2    rgba(236,231,218,.16)
--text      #ECE7DA   月白·主文
--dim       #8B8F9A   次文
--faint     #565A66   三级 / 占位
--gold      #C8A86A   描金·强调 / 选中 / 关键数
--cinnabar  #9B2226   朱砂·点睛 / 警示 / 钤印
--dai       #3C4A60   黛蓝·图表 / 山
--qing      #5F8A82   青·辅助
```
品牌色克制：金=强调，朱砂=点睛（一屏≤1–2 处），黛/青=数据可视化。

**字**
- 显示 / 标题 / 关键数：`Songti SC` Light（衬线，editorial 高级感），字距 +1~3。
- 正文 / UI / 标签：`PingFang SC` / system sans。
- 字阶（pt）：48 hero数字 / 28 页面标题 / 19 卡标题 / 15 正文 / 13 次文 / 11 标签。行高 1.6–1.9。

**间距 / 形 / 高度**
- 间距步进 4：4 / 8 / 12 / 16 / 20 / 24 / 32。
- 圆角：卡 20–22 · 输入 12 · 按钮 全圆/14 · App图标 22 · 头像/球 50%。
- 高度（阴影）：卡 `0 10 30 rgba(0,0,0,.5)`；浮层 `0 24 60 rgba(0,0,0,.6)`。
- 光向统一左上→右下（与天体一致）；可叠 5% 噪点 grain 增厚度。

**图标**：v10 自绘，24 网格 / 1.75 描边 / 圆角端；选中描金。

**动效原则**：慢、流体、有"天体/墨"的呼吸感；转场用 fade + 轻位移，避免弹跳。

**组件原语**（待逐一定义）：主按钮(金描边/实底) · 次按钮(幽灵) · 卡 · chip/标签 · 分隔线 · **分数表盘** · **命盘可视化** · 列表行 · 底部 sheet · toast · tab bar · nav bar。

## 4. 内容 / 数据模型（让 UI 有据可依）

```
Reading {
  app, palace(命/相/卜),
  subject,            // 生辰 or 问题 or 照片
  verdict:{ score?:0-100, label },   // 如 84 / "中庭得位"
  visual,             // 命盘 / 卦象 / 山水 / 蝶 …
  sections:[ {key, title, body, tone:吉/平/慎} ],  // 总览 / 分项 / 流年 / 宜忌 / 建议
  shareSummary,       // 一句可分享的金句
  ts
}
Almanac { date, 干支, 节气?, 宜:[], 忌:[], 吉时, 冲煞 }
Match { a, b, score, 项:[{名,分}], 宜忌 }
User { 生辰档案[], 收藏[], 历史[], vip }
```

**语气 Voice**：古典 + 克制 + 一丝神秘 + 可信；短句、文白相间；忌浮夸营销腔、忌恐吓。
例：「中庭鼻准丰隆，主三十至五十财官有成；唯眉宇稍促，宜舒展心境。」

## 5. 体验铁律

- 深色为主、留白足、信息分层（hero → 概览 → 分项）。
- 一屏一焦点；朱砂只点睛。
- 玄学要"可信而非吓人"：给依据、给建议、给分寸。
- 关键数 / 卦象用衬线 + 描金做仪式感。
- 可访问性：正文对比≥4.5:1，触控目标≥44pt。

## 6. 需要设计的场景清单

> 标 **P0** = 英雄时刻，先做；P1 次之；P2 补全。每条列「目的 + 关键组件」。

### A. 启动 / Onboarding
- **A1 启动页 / Splash**（P1）：古字字标 / 天体一瞬。
- **A2 生辰录入**（P0）：年月日时(农/公历切换) + 地点 + 性别；用 v10 日期/时辰/地点图标；底部主按钮"起盘"。
- **A3 问卜输入**（P1，卜宫）：文字/语音问题 + "摇卦"动作。

### B. 首页 / Home
- **B1 今日卡 + 轮播**（P0）：顶部今日运势 hero；横向 card list（昨夜之梦 / 今日 / 本周姻缘 / 宜忌），中卡浮起（见 v7）。
- **B2 三宫入口宫格**（P1）：命/相/卜 分组的 App 入口（用 v9 主图标）。
- **B3 快捷起卦条**（P2）。

### C. 命盘 / Chart
- **C1 命盘可视化**（P0）：星座般发丝连线的命盘（见 v7 report 内小图放大版）；12 宫可点。
- **C2 宫位详情**（P1）：点宫位 → 该宫解读 sheet。

### D. Reading Report（P0，核心交付物）
- hero（主图标/球 + app + 日期）→ 分数表盘 + 一句判词 → 命盘/卦象可视化 → 分项 sections（总览/分项/流年/宜忌/建议，发丝线分隔）→ 操作条（收藏/分享/再卜）。参考 v7。

### E. 分享卡 / Share（P0）
- **E1 金句卡**：球/母题 + 一句判词 + 钤印（见 v7）。
- **E2 分数卡**：大分数 + 标签。
- **E3 命盘卡**：命盘可视化 + 关键词。
- 规格：1080×1512（竖）/ 1080×1080（方）；底部 app 水印 + 朱砂印。

### F. 历 / 黄历（P1）
- **F1 今日**：宜/忌大字 + 节气 + 干支 + 吉时 + 冲煞。
- **F2 月视图**：节气标注的月历。

### G. 合婚 / Match（P1）
- 双人生辰输入 → 合婚分 + 分项条 + 宜忌。

### H. 解梦 / Dream（P1）
- 输入(文字/语音) → 释义卡（象征 / 吉凶 / 提点）。

### I. 探索 / Discover（P2）
- 知识 / 文章 feed；卡片式。

### J. 我的 / Profile（P1）
- 生辰档案、收藏、历史、VIP 入口。

### K. 会员 / Paywall（P1）
- 权益对比 + 价格 + 主 CTA；克制不逼单。

### L. 状态（P1）
- 空状态 / 加载（天体/墨呼吸动效）/ 错误 / 无网。

### M. 系统（P2）
- 通知中心 / push 样式、设置页。

### N. 组件库（P0，贯穿）
- tab bar · nav bar · 主/次按钮 · 卡 · chip · **分数表盘** · **命盘可视化** · 列表行 · 底部 sheet · toast。

## 7. 建议下手顺序

1. **N 组件原语**（按钮 / 卡 / 分数表盘 / 命盘可视化）——后面所有页都复用。
2. **D Reading Report** + **E 分享卡**——产品最高光、最常被截图传播。
3. **B1 今日卡 + 轮播**——首屏留存。
4. **A2 生辰录入**——转化漏斗第一步。
5. 其余按 P1 / P2 补全。
