Filter Bake · 离线烘焙

运行时 feTurbulence + feDisplacementMap 在单页 50 + 图标同框时 Safari 帧率掉到 30 以下，RN 上更慢。 方案：tooling/icon-bake/bake.ts 用 paper.js 把 displacement 形变烘焙进静态 path，运行时零 filter 调用。 页面顶部 inkRough filter 仍保留作为编辑期预览，正式构建时由 bake 步骤替换。

ID Hash · 防碰撞

v12 的 id="inkRough" 全局共享（所有 glyph 用同一份 filter，OK）。 但 sphere 变体需要 每个实例的 gradient 参数不同，如果都叫 id="hi"，浏览器只渲染第一个。 下方 sphere 生成器用自增 counter 给每个实例生成 id="h{n}"。RN/SVGR 出包时用 nanoid。

设计原则

v13 在 v12 4 条原则上追加 4 条工程 + 矩阵原则。

12 principles
风格原则（v12）整套图标只允许一种主墨色；质感来自纸、墨边、留白和笔触。
造型原则（v12）八字从「柱」出发，紫微从「宫格」出发，时间从「轨道/节点」出发。
细节原则（v12）只保留能帮助识别的细节。墨迹毛边是材质，不承担语义。
落地建议（v12）P0 优先：命盘、八字、紫微、五行、四柱、日主、命宫、主星、大运。
性能原则（v13）所有 turbulence/displacement 上线前必须 bake 成静态 path。运行时 SVG filter 调用必须 = 0。
命名原则（v13）所有 SVG def id 用 nanoid，禁用语义命名（如 id="hi"），避免组件复用碰撞。
层级原则（v13）App Store logo（sphere）和 in-app glyph（ink stroke）是两个独立子系统，禁止互相复用形状。
矩阵原则（v13）8 个 sphere 变体只动 4 个杠杆，不加文字/silhouette/外框。轻则保持家族识别，重则破坏品牌一致。
