# ADR-0017 · V1.5 Wave — face / coin / dream Redesigned per Doctrine v2

> **Status**: ACCEPTED · 2026-05-31
> **Implements**: ADR-0015 Doctrine v2 §"Conceptual Uniqueness Checklist" for 3 additional apps
> **Supersedes**: ADR-0016 §"Archive face/coin/dream" (reversed same-day; see ADR-0016 revision summary)
> **Re-sequenced by**: [ADR-0019](0019-v1-wave-narrowed-cycle-feng-yuan.md) — V1.5 wave activation now sits **after MingPan + numerology resume** (V1 cycle/feng/yuan → V1.1 MingPan + numerology → V1.5 Coincast / FaceRead / DreamRead), not directly after V1. The 6-axis scorecards + sprint plans + reviewer-notes templates below are unchanged; only the activation window shifts.

## Context

The original V1 plan (per ADR-0015) covered 4 apps: cycle, feng, yuan v2,
MingPan. Three additional apps (face-oracle, coin-cast, dream-oracle) were
provisionally archived per ADR-0016.

Several hours after that archive decision, the user observed:

> "I see Apple Store also has face reading and oracle apps, so could the other
> four apps also ship? Please check pass odds and adjust the plan."

This observation triggered a re-evaluation using the same Conceptual Uniqueness
Checklist from ADR-0015. Result: the 3 apps **can pass review** when redesigned
per Doctrine v2, scoring 5.5-6/6 on the uniqueness axes. Established App Store
precedent confirms the categories are not blocked at the feature level.

The corrected plan is a **V1.5 wave** — a second wave of submissions following
V1 validation, allowing for measured rollout that reduces spam-family detection
risk while preserving the full 7-app product matrix.

### V1.5 trigger criteria

V1.5 dev work begins when ALL of:

1. ✅ V1 W1 cycle approved + live in App Store
2. ✅ ≥ 1 additional V1 app (feng / yuan / MingPan) also approved
3. ✅ ≥ 30 days of cycle production telemetry (subscription conversion ≥ 1% baseline)
4. ✅ Doctrine v2 anti-spam framing empirically validated (no V1 app rejected
   for category-saturation reasons)
5. ✅ Engineering bandwidth allows (typically: 1-2 engineers freed up from V1 polish work)

**If any V1 app is rejected for 4.3(b)**: V1.5 wait extends until appeal-cycle
completes. **If 2+ V1 apps are rejected**: V1.5 is paused indefinitely; escalate
to SPAM-11 PWA fallback for non-passing apps.

### V1.5 ordering rationale

W5 = **Coincast first** (not Face / Dream)
- I-Ching apps have the highest established App Store precedent
- Pass odds 80-85% (highest of 3)
- Builds publisher credit for the 4.3(b) categories of W6 and W7
- Coincast's "study tool" framing is least disputable

W6 = **FaceRead second**
- 4.3(b) cited "palm reading" — needs Coincast precedent to land
- 75-80% pass odds
- Family face lineage is unique enough to clear category saturation

W7 = **DreamRead last**
- "Dream interpretation" has many established precedents but "Oracle" suffix
  raised flag in SPAM-17 (handled by rename to DreamRead)
- 75-80% pass odds
- Strongest psychological-framework angle for differentiation

### V1.5 redesign work scope per app

Each V1.5 app needs roughly the same redesign pattern:

1. **Strip narrative**: remove "your dream means X" / "your face shows Y" copy
2. **Add structural output**: deterministic features → algorithmic visualizations
3. **Add classical reference layer**: tap-to-deep-read for every feature
4. **Add daily-utility hook** (where appropriate):
   - Coincast: journal + study path
   - FaceRead: time-series + family lineage
   - DreamRead: pattern dashboard + sleep correlation
5. **Add 4-locale content**: ~30-50K words per app
6. **Add widget × 3 + Watch complication**: leverage `packages/widget-kit-ios`
7. **Reframe ASO**: utility/educational tone, no "oracle/divination/palm reading"
8. **Anti-spam screenshot design**: utility-anchored visuals only

**Per-app investment**: 4-5 weeks (slightly less than V1 apps because shared
infra and patterns from V1 are reused).

## Consequences

### Positive

- **7-app product matrix** — full coverage of Chinese metaphysics categories
- **Validated approach** — V1.5 follows V1 validation, not parallel risk
- **Engineering leverage** — V1 infra (widget-kit + classical-refs + content
  patterns) directly reusable for V1.5
- **Revenue scaling** — 3 more subscription products + Universe Bundle V1.5
  enables higher matrix ARPU
- **Brand depth** — single publisher (UseONE, LLC) with comprehensive product
  line signals serious player, not casual horoscope shop

### Negative

- **30-week total to full matrix** — longer commitment vs ship-and-iterate
  approach
- **V1.5 dependency on V1 success** — 2 V1 rejections → V1.5 paused indefinitely
- **Content authoring burden** — 30-50K words × 3 apps × 4 locales =
  ~400-600K words across V1.5 (large translation + native-review cost)
- **Engineering team scaling pressure** — 7 apps in production = 7 apps to
  maintain, debug, deploy

### Risks + mitigations

| Risk | P | Impact | Mitigation |
|---|---|---|---|
| V1.5 triggers don't fire (V1 apps fail) | 15% | V1.5 indefinitely paused | Triggers documented; SPAM-11 PWA fallback available |
| Engineering bandwidth runs out | 30% | V1.5 delays 2-3 months | Sprint plans designed with 1-engineer assumption; can scale |
| Coincast / FaceRead / DreamRead rejected for 4.3(b) | 15-25% per app | One app delayed 4-6 weeks per appeal | Per-app reviewer notes template (per ADR-0015) addresses pre-emptively |
| Universe Bundle V1.5 user confusion (vs V1) | 20% | Lower bundle attach | Grandfather V1 subscribers; clear V1 → V1.5 migration UX |
| Content authoring takes > 30 weeks | 40% | V1.5 delays 6-8 weeks | AI-drafted + native review parallelism; start content authoring in V1 W3 |

## Validation criteria (when to revise this ADR)

**Revise if**:
- V1.5 W5 Coincast rejected on first submission with "saturation" reason → re-examine 6-axis scorecard
- Universe Bundle V1.5 attach rate < 10% by month 3 of W7 ship → pricing wrong
- Engineering bandwidth chronically insufficient → reduce V1.5 to 2 apps

**Re-confirm if**:
- All 3 V1.5 apps cleared review within 6 months of V1 W1 ship
- Universe Bundle V1.5 LTV > $500 per user
- Cross-app discovery → V1.5 install rate > 25%
